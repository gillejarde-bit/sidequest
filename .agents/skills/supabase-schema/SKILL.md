# Supabase Schema Skill

## Migration Conventions
- Use `snake_case` for all tables and columns.
- Always use `uuid` for Primary Keys, generated by `gen_random_uuid()`.
- Use `timestamptz` for all date/time columns (e.g., `created_at timestamptz default now()`).
- Always write reversible migrations (Down migrations must be included or instructions to rollback).

## RLS Policy Patterns
- Always enable RLS on every table: `ALTER TABLE table_name ENABLE ROW LEVEL SECURITY;`
- Policies should check `auth.uid()` for row ownership.
- Public read access for social data, restricted write access to owners.

## PostGIS Query Patterns
- Use `geography(Point, 4326)` for all location columns.
- Index: `CREATE INDEX idx_locations_geo ON locations USING GIST (geo);`
- Nearby query: `ST_DWithin(geo, ST_Point(lng,lat)::geography, meters)`
- Distance query: `ST_Distance(geo, ST_Point(lng,lat)::geography)`

## Type Regen
- Command: `pnpm supabase gen types typescript --linked > src/types/database.types.ts`
- Must be run after EVERY migration.
